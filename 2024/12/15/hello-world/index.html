📖发布文章

你需要在发布文章时，按照以下格式写标头：

  • title: 一个简约扁平化的Hexo静态主题博客
  • comments: false
  • categories: 项目案例
  • aubot: Cange-Q
  • aubot_link: ‘xxxx’
  • tags: Hexo Quiet主题 静态主题
  • excerpt:简约大方
  • toc: false
  • date: 2020-11-03 20:33:36
  • cover: ‘xxxx’

解释

  • title：文章标题。
  • categories：分类（最好只写一个）。
  • tags：标签可以有多个。
  • excerpt：文章描述。
  • toc：是否显示文章目录（默认值为 false）。
  • date：创建日期。
  • cover：文章缩略图（不填则使用默认的）。
  • comments：是否开启评论。
  • aubot：作者名称，未设置则默认为配置文件中的名称。
  • aubot_link：点击作者名称时跳转的地址，默认跳转到关于我的页面。

🏆主题配置

进入主题根目录下的 themes 文件夹中的 Quiet 目录，找到 _config.yml 配置文件，我们可以在里面设置网站的标题、各种 Logo 图标等。

添加友情链接

在配置文件中，找到 linksList，可以按照示例添加你的友情链接。

💌评论配置

本主题使用 gitalk 评论插件，它基于 Github Issue 和 Preact 开发。你只需要在 Quiet 主题文件夹中的 _config.yml 配置文件的 gitalk 部分进行配置。

首先，你需要选择一个公共 GitHub 存储库（已存在或创建一个新的）来存储评论，并创建 GitHub 应用。如果没有应用， 点击这里申请，Authorization callback URL 填写当前使用插件页面的域名。

配置项：

  • clientID String：GitHub 应用的 Client ID。
  • clientSecret String：GitHub 应用的 Client Secret。
  • repo String：GitHub 存储库。
  • owner String：GitHub 存储库的所有者，可以是个人或组织。
  • admin Array：GitHub 存储库的所有者和合作者（对这个存储库有写权限的用户）。
