欢迎来到 Hexo！这是一篇详细的指南，帮助你快速上手并配置Hexo博客。如果你在使用过程中遇到任何问题，可以参考 官方文档 或在 GitHub 上寻求帮助。

快速开始

创建新文章

要创建一篇新文章，使用以下命令：

$ hexo new "文章标题"

文章文件将生成在 source/_posts/ 目录下，你可以使用Markdown语法编辑内容。

更多信息：写作

启动本地服务器

启动Hexo本地服务器，实时预览博客内容：

$ hexo server

启动后，访问 http://localhost:4000 查看效果。

更多信息：服务器

生成静态文件

生成静态文件以便部署：

$ hexo generate

生成的静态文件将存放在 public/ 目录中，你可以将其部署到GitHub Pages、Netlify等平台。

更多信息：生成

发布文章

发布文章时，请按照以下格式编写文章头部信息：

title: 文章标题
comments: false
categories: 分类名称
aubot: 作者名称
aubot_link: '作者链接'
tags: 
  - 标签1
  - 标签2
excerpt: 文章摘要
toc: false
date: 2024-12-15 20:00:00
cover: '文章封面图链接'

参数说明：

  • title：文章标题。
  • comments：是否开启评论（默认 false）。
  • categories：文章分类（建议只填写一个）。
  • aubot：作者名称，未设置时使用配置文件中的默认值。
  • aubot_link：点击作者名称时跳转的链接，默认跳转到“关于我”页面。
  • tags：文章标签，支持多个标签。
  • excerpt：文章摘要。
  • toc：是否显示文章目录（默认 false）。
  • date：文章发布日期。
  • cover：文章封面图链接，留空时使用默认图片。

主题配置

Hexo主题的配置文件位于 themes/Quiet/_config.yml。你可以在此文件中设置网站标题、Logo、图标等。

添加友情链接

在配置文件中找到 linksList，按以下格式添加友情链接：

linksList:
  - name: 友链名称
    link: 友链地址
    avatar: 友链头像链接
    desc: 友链描述

评论配置

本主题使用 Gitalk 评论插件，基于GitHub Issue实现。你需要在 _config.yml 中配置以下参数：

  1. 创建GitHub应用
    如果没有GitHub应用，请先 申请。在 Authorization callback URL 中填写你的博客域名。

  2. 配置Gitalk
    在 _config.yml 中找到 gitalk 部分，填写以下信息：

gitalk:
  clientID: '你的GitHub应用Client ID'
  clientSecret: '你的GitHub应用Client Secret'
  repo: '存储评论的GitHub仓库名称'
  owner: 'GitHub用户名或组织名'
  admin: 
    - 'GitHub用户名'

参数说明：

  • clientID 和 clientSecret：GitHub应用的凭证。
  • repo：用于存储评论的GitHub仓库。
  • owner：仓库所有者。
  • admin：仓库管理员列表（拥有写权限的用户）。

其他功能

  • 文章封面图：建议使用高质量的图片链接，提升视觉效果。
  • 文章摘要：简洁明了地概括文章内容，吸引读者点击。
  • 标签与分类：合理使用标签和分类，方便内容管理。
